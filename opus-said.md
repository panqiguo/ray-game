这是一个非常好的问题，也是很多设计者在从"概念文档"走向"可玩体验"时都会撞上的墙。让我认真分析一下。

# 问题诊断：为什么感觉像"功能框架"而不是"游戏"

你描述的感受可以拆成两个具体问题：

**城市整备：** "没有分配的必要感，没有策略感"
→ 因为缺乏**资源压力**和**信息不对称**。如果什么都能做、什么时候做都一样，选择就不存在。

**任务推进：** "只是在走流程"
→ 因为缺乏**局面感知**和**动态变化**。如果每个行动点都是孤立的"选一个选项→看结果"，那就是在点菜单，不是在闯关。

根源在于：**你的系统有"机制"但还没有"张力"。** 张力来自于三样东西——稀缺、信息差、后果的连锁反应。

# 应该学习的游戏骨架

我按你的两个模式分别推荐，每个都说明**具体学什么**：

## 城市整备：学谁？

### 1. 《Citizen Sleeper》 — 最直接的参考

```
你的设计                          Citizen Sleeper
─────────────────────────────────────────────────
每天抽5张手牌                     每天掷5个骰子（1-6）
用手牌做城市行动                  把骰子分配到行动槽
手牌花色影响结果                  骰子点数影响成功率
有限天数                          有多条时间线同时倒计时
```

**它做对了什么：**

- **骰子分配就是痛苦的选择。** 你掷出 [1, 2, 3, 5, 6]，6 只有一个，放在哪里？高风险高回报的任务？还是保命的食物获取？**每一颗骰子的分配都是一次取舍。**

- **多条时间线同时跑。** 不是只有一个"乌鸦的时间"在倒计时。你同时面对：房租要到期了、一个NPC三天后要离开空间站、另一个NPC需要你帮忙否则会死。你**不可能全部照顾到**。

- **行动的结果会改变明天的选项。** 帮了A，B的任务线就会变化。不是孤立的"做任务拿奖励"。

**你应该从中偷的东西：**

> 城市整备之所以有意义，不是因为"有很多地方可以去"，而是因为**你今天只能去2-3个地方，但有5个地方都需要你**。

具体到你的设计：
- 乌鸦时间钟在倒计时，但你需要钱买烟卷，需要去诊所清污染，需要去收集线索降低任务难度，还需要打工赚钱——**一天5张牌不够做所有事**
- 某些行动有**窗口期**：线人只在第2天出现，走私商第3天就跑路

---

### 2. 《Tainted Grail》（桌游）— 地图探索的时间压力

**它做对了什么：**

- 地图上的灯塔会熄灭。灯塔熄灭后，对应区域就无法进入。你必须在探索新区域和维护已有区域之间分配资源。
- **你永远在"推进"和"维护"之间撕裂。**

**你应该从中偷的东西：**

> 城市地图不应该是静态菜单。随着天数推进，某些地点应该**变化或消失**。

比如：
- 第1天黑市有烟卷卖，第3天黑市被查封（Heat太高）
- 第2天线人在酒吧，第3天线人被杀（你没及时去）
- 诊所第1天免费，第2天开始收费（因为你引起了注意）

---

### 3. 《Signs of the Sojourner》 — 卡牌作为社交语言

```
这个游戏的核心：
你的牌组 = 你的沟通方式
你去不同的地方，牌组会被改变
去太多粗暴的地方，你就失去了温柔沟通的能力
```

**你应该从中偷的东西：**

> 牌库不只是"资源池"，牌库的构成本身就是叙事。

你已经有了"污染"的概念，但可以更进一步：
- 如果玩家反复使用【强硬】解决问题，牌库中【强硬】牌会逐渐增多（侦探变得越来越暴力）
- 如果玩家反复受伤，物理负面牌堆积，侦探的"潜意识"真的被创伤填满
- **牌库的变化就是角色弧光的机械表达**

## 任务推进：学谁？

### 4. 《Slay the Spire》 — 不是学战斗，是学"地图即决策"

你的任务模式现在是线性的：外围→走廊→冷库，每个区域里有几个行动点。这就是"走流程"的感觉来源。

**Slay the Spire 做对了什么：**

```
       [?]
      /   \
   [怪]   [商店]
    |       |
   [?]   [篝火]
      \   /
      [BOSS]
```

- 不是线性的，是**有分支的路径选择**
- 每个节点你都在权衡：我现在血量低，走商店还是赌问号？
- **地图本身就是决策层**，不只是"走到下一个房间"

**你应该从中偷的东西：**

任务地图不必是线性三段。即使只有三个区域，区域内的行动点也可以有**顺序依赖和互斥关系**：

```
外围:
  铁丝网 ──→ [走廊入口A: 正门]
  排水管 ──→ [走廊入口B: 地下]     ← 只能选一条路进走廊
  
走廊:
  (从正门进) → 恶犬 → 灯控室 → 冷库
  (从地下进) → 积水通道 → 直达冷库后门（但跳过灯控室）
```

**选择路径 = 选择你将面对什么问题 + 放弃什么机会。**

### 5. 《Blades in the Dark》（TTRPG）— 任务中的动态压力

你已经借鉴了进度钟，但 Blades 最核心的不是进度钟，而是**"position & effect"（处境与效果）**的动态变化。

**它做对了什么：**

- 每个行动不只是"成功/失败"，而是你当前处于什么**处境**（controlled/risky/desperate）
- 处境会因为前一个行动的结果而**连续变化**
- 你搞砸了一次，不是"扣血"然后继续，而是**整个局面恶化**——下一个行动自动从更差的处境开始

**你应该从中偷的东西：**

> 行动点之间不应该是孤立的。前一个行动的结果应该**改变后续行动的条件**。

具体实现：

```python
# 不是每个行动点独立配置难度
# 而是有一个"局面状态"影响所有后续行动

class ScoreState:
    alert_level: int      # 来自警报钟
    noise_made: bool      # 是否制造了噪音
    guard_down: bool      # 是否解决了守卫
    lights_off: bool      # 是否关了灯
    
    def get_difficulty_modifier(self, action_id):
        mod = 0
        if self.lights_off and action_id in ["sneak_past", "ambush"]:
            mod += 1      # 关灯后潜行类行动更容易
        if self.noise_made:
            mod -= 1      # 制造噪音后所有行动更难
        if self.alert_level >= 3:
            mod -= 1      # 高警报全局惩罚
        return mod
```

这样玩家的感受就从"逐个解决行动点"变成了"**我在经营一个不断变化的局面**"。

### 6. 《Inscryption》 — 学它的"仪式感"

Inscryption 的卡牌机制本身不复杂，但它让每一次出牌都有**重量感**：
- 天平秤会物理倾斜
- 对面的GM会对你的出牌做出反应
- 牺牲卡牌时有视觉和音效的仪式

**你应该从中偷的东西：**

> 不是机制让游戏有感觉，是**反馈**让游戏有感觉。

你的侦探独白系统是对的方向，但可以扩展：

```
掷骰前:
  "手里只剩一张牌了。就这一次机会。"
  
掷骰结果为代价成功:
  "门开了。但那声响整条走廊都听见了。"

硬撑时:
  "你咬紧牙关。眼前发黑。再来一轮。"
  (屏幕边缘轻微变暗/扭曲)

Stress接近上限:
  (手牌开始轻微抖动)
  (侦探独白变得碎片化)
```

# 核心结论：你的设计缺什么

你的设计不缺机制，缺三样东西：

## 1. 竞争性需求（城市整备的张力来源）

```
当前: 你有5张牌，城市有5个地点，你可以慢慢做
应该: 你有5张牌，城市有5个地点，但——
      - 乌鸦时间在倒计时
      - 你的Health只剩3，需要去诊所
      - 但你没钱，需要先打工
      - 但线人明天就走，今天不去就没线索
      - 没有线索进任务会更难
      → 你只能做其中2-3件事。选。
```

## 2. 行动之间的因果链（任务推进的局面感）

```
当前: 行动点A → 结果。行动点B → 结果。行动点C → 结果。
应该: 行动点A的结果 → 改变行动点B的条件 → 影响行动点C是否存在
      
例:
  关灯(成功) → 恶犬行动难度-1，但头目提高警惕+1
  关灯(失败) → 短路，警报钟+2，但意外发现通风管道(新路径)
  没去灯控室 → 恶犬和头目维持原始难度，但你省了一张牌
```

## 3. 状态恶化的感官表达（情境体验的来源）

```
Stress 0-2:  正常UI，正常独白
Stress 3-5:  手牌轻微抖动，独白变短促，BGM加入心跳底噪
Stress 6-7:  屏幕边缘暗角，卡牌花色文字偶尔闪烁/模糊
Stress 8(满): 强制塞入负面牌时，屏幕短暂闪白，独白变成碎片
              "不行了——" "再撑一下——" "别想那些——"
```

这不需要复杂的着色器。raylib 里一个半透明黑色矩形画在屏幕边缘就是暗角，卡牌坐标加个 `sin(time * frequency) * amplitude` 就是抖动。成本极低，效果极大。

# 行动建议

在写代码之前，先做一件事：

**用纸牌模拟一个完整的3天城市 + 任务流程。**

- 拿一副扑克牌，红桃/方块/黑桃/梅花代表四花色
- 在纸上画城市地图和任务地图
- 自己跟自己玩一遍
- 记录每个决策点你的感受："这里我有选择吗？""这个选择痛吗？""前一个行动影响了这个行动吗？"

如果纸面原型都不好玩，代码实现也不会好玩。如果纸面原型在某个时刻让你犹豫了、纠结了、后悔了——那个时刻就是你的游戏真正存在的地方。
