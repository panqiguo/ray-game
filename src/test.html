一句话概括

《苏丹的游戏》的核心不是“一个主线任务系统”，而是： 一个每周强制抽取死亡任务的生存框架， 外面包着大量角色、资源、事件、路线和声望系统。

也就是说，玩家不是在自由地做任务，而是在 “七天内必须处理苏丹卡，否则死亡” 的压力下，利用世界里不断出现的机会，把资源转化成解决方案。 官方 Steam 页面也直接把它描述为：每周抽取 Carnality、Extravagance、Conquest、Bloodshed 四类卡之一，七天内满足要求，否则被处刑。:contentReference[oaicite:0]{index=0}

一、最底层：所有东西都是“卡”

这个游戏最重要的抽象是：世界里的东西都可以被做成卡牌。

  • 主角是卡。
  • 随从是卡。
  • 装备是卡。
  • 书籍是卡。
  • 情报是卡。
  • 思潮是卡。
  • 金币、住宅、毒药、武器、苏丹卡也都可以被当成卡处理。

BWiki 的新手指南也明确说，手牌包括主角、随从、武器装备、情报、物品和苏丹卡；角色属性可以通过读书和事件增加，装备则给角色增加属性。:contentReference[oaicite:1]{index=1}

所以它的底层不是传统 RPG 的： 

角色面板 + 背包 + 任务日志 + 地图事件

而更接近： 

一切都是可被放入事件槽位的对象卡。

这点非常关键。因为一旦所有东西都是卡，游戏就可以用同一套规则处理：

  • 谁参与这件事？——放角色卡。
  • 用什么办法？——放物品卡、装备卡、情报卡。
  • 付出什么代价？——消耗金币、消耗人、消耗苏丹卡。
  • 触发什么剧情？——卡牌组合满足条件后生成仪式。

二、核心循环：抽苏丹卡 → 找销卡方法 → 活下去

这个游戏最核心的循环可以写成： 

抽到苏丹卡
→ 产生七天倒计时
→ 世界中寻找/制造销卡机会
→ 消耗角色、资源、道具、声望、事件机会
→ 销掉苏丹卡
→ 获得新资源/新路线/新后果
→ 下一张苏丹卡继续压迫你

BWiki 明确说，持有苏丹卡超过七天会被处刑，游戏最核心玩法就是“通过各种手段方式消除苏丹卡，以达成胜利条件”。:contentReference[oaicite:2]{index=2}

所以你说的“每 7 天给你一个任务”是对的。但更准确地说，它不是普通任务，而是：

周期性死亡压力。

这会把玩家所有行动都压缩到一个问题里： 

我现在做这件事，是为了马上销卡？
还是为了投资未来，让之后更容易销卡？
还是为了推进某条终局路线？

三、苏丹卡不是单一任务，而是“目标类型”

苏丹卡的聪明之处在于：它没有只写一个固定解法，而是声明一种目标类型。

苏丹卡类型 表层要求 实际设计作用
纵欲 满足欲望 迫使玩家使用关系、身体、妃子、牺牲、替代方案
奢靡 大量花钱 迫使玩家建设经济、消费、投资住宅、制造金钱缺口
征服 完成支配/远征/战争相关行为 迫使玩家培养战力、随从、装备、军队或特殊解法
杀戮 献上死亡 迫使玩家牺牲角色、制造替罪羊、寻找道德/政治解法

每类卡又有品质高低。BWiki 提到，品质越低越容易消除，高品质卡需要更多准备，杀戮和征服通常更难处理。:contentReference[oaicite:3]{index=3}

因此，苏丹卡不是： 

任务：去 A 地点杀 B。

而是： 

任务类型：你必须在七天内完成一次“杀戮”。
可行解法：牺牲某人、利用政治事件、触发特殊剧情、用某条路线替代、用某个角色承担代价……

这就是它产生复杂度的地方：目标是固定的，但解法空间是开放的。

四、事件系统：不是地图任务，而是“仪式槽位”

游戏里的事件通常叫“仪式”。它们本质上是一个个可以放卡的处理器。 

仪式 = 条件 + 槽位 + 时间成本 + 检定 + 结果

一个仪式大概可以理解成： 

{
  id: "治理家业",
  trigger: "常驻",
  slots: ["贵族或随从"],
  duration: 1,
  test: "智慧 + 社交",
  result: {
    success_3: "+5 金币",
    success_2: "+3 金币",
    success_1: "+2 金币",
    success_0: "+1 金币"
  }
}

BWiki 对“治理家业”的描述就非常接近这个结构：放入角色后，根据智慧+社交总和决定骰子数量，结算时根据成功数获得不同数量金币。:contentReference[oaicite:4]{index=4}

这类仪式有几种来源：

  • 常驻仪式：例如治理家业、浴场、书店、上朝。
  • 时间触发仪式：某些天数、倒计时、阶段后出现。
  • 前置事件触发仪式：完成 A 后出现 B。
  • 计数器触发仪式：某种状态累积到一定值后出现。
  • 卡牌主动触发仪式：把书、金币、情报、角色、苏丹卡拖进“俺寻思”。

BWiki 也说明，仪式会根据时间、浴场判定、前置事件、计数器累积等条件出现；事件有持续回合数，也有执行所需回合数，期间放入的角色和物品不可移动。:contentReference[oaicite:5]{index=5}

五、“俺寻思”是事件发现器，也是组合实验器

“俺寻思”的功能很重要。它不是普通菜单，而是一个“把卡投入系统，看看能不能触发事件”的机制。 

卡牌 → 俺寻思 → 可能生成仪式

例如：

  • 放书 → 触发读书/学习事件。
  • 放金币 → 触发随机打工事件。
  • 放情报 → 触发情报事件。
  • 放角色 → 触发特殊角色事件。
  • 放苏丹卡 → 触发对应销卡事件。

这些例子在 BWiki 里都有列出。:contentReference[oaicite:6]{index=6}

这个系统的设计意义是：

它让玩家觉得世界不是一个任务列表，而是一个可以用资源试探、组合、发现的黑箱。

你说的“密密麻麻的不确定的东西”主要就来自这里：玩家不是清楚地知道所有路线，而是通过卡牌组合、临时事件、随机事件和前置条件逐渐摸出结构。

六、资源系统：所有资源都不是单纯奖励，而是未来解法

你可以把游戏里的资源分成几类：

资源类型 作用 设计意义
金币 买书、买装备、投资住宅、支付事件成本 通用燃料
书籍 提升角色属性 长期成长
装备 临时提高属性，改变某人能完成的任务 可调度能力
情报 增加检定属性、重骰、触发事件 风险缓冲与路线钥匙
思潮 用于政治/路线/角色加入等事件 路线材料
随从 执行仪式、提供属性、参与路线 行动单位与叙事单位
声望 触发事件、影响分支、影响结局 长期身份倾向
住宅/家产 周期性产出金币或提供其他收益 经济引擎

这套资源系统的关键不是“拿奖励很爽”，而是： 

资源 = 未来解决苏丹卡或终局路线的可能性。

例如金币不是钱本身，而是：

  • 以后处理奢靡卡的准备；
  • 购买装备的能力；
  • 读书提升属性的入口；
  • 逃亡、谋反、路线事件的成本；
  • 扩大住宅，形成长期收益。

所以玩家做“治理家业”“买书”“浴场情报”这些行为，看似是日常经营，实际是在给未来的死亡任务准备解法。

七、行动经济：最重要的资源其实是“回合”和“人手”

《苏丹的游戏》里最紧张的不是金币，而是：

  • 还剩几天；
  • 这个事件要占用几回合；
  • 谁被锁在事件里；
  • 这个人现在能不能被拿去处理另一件事。

事件面板的持续回合和执行回合非常重要，因为角色、物品被放入后会被锁定。BWiki 明确说，事件有持续回合数，执行事件也需要回合数，期间放入的角色物品不可移动。:contentReference[oaicite:7]{index=7}

这意味着游戏的真实问题经常不是： 

我有没有能力做这件事？

而是： 

我有没有时间做这件事？
我把这个人派出去后，另一个销卡事件怎么办？
我现在投资未来，会不会来不及处理手上的苏丹卡？

这就是它和普通 RPG 的区别。普通 RPG 中，角色成长常常是“变强”。这里的成长更多是：

增加调度余地。

八、检定系统：属性不是单纯数值，而是“概率门槛”

你前面说的骰池理解是对的。这个游戏的检定大致是： 

投入角色/物品
→ 计算相关属性总和
→ 得到骰子数量
→ 掷骰
→ 根据成功数量决定结果档位

BWiki 对检定的说明是：骰子数量由角色属性、是否产生对抗、放入物品决定；对抗会按对手属性扣减骰子；合适物品可以跳过检定直接成功，情报等消耗品也可以增加属性和骰子数量。成功数量越多，仪式结果通常越好。:contentReference[oaicite:8]{index=8}

所以属性成长在这里有三层意义：

  1. 提高成功概率：骰子多，成功数期望更高。
  2. 打开事件门槛：某些事件要求特定属性或特定角色能做。
  3. 提高收益档位：不是简单成功/失败，而是成功数越高，奖励越好。

这也是为什么装备和情报很重要。它们不是普通加成，而是可以临时改变某个事件能不能过、能不能拿到高档奖励。

九、固定与随机：它不是纯随机，而是“固定压力 + 随机机会”

你对固定和随机的感觉很准确。可以这样拆：

层级 固定性 作用
苏丹卡倒计时 强固定 稳定制造压力
四类苏丹卡目标 半固定 决定当前生存问题类型
常驻仪式 固定 给玩家基本经济与信息来源
角色线/路线线 前置固定 玩家知道可以推进，但未必知道完整条件
浴场/情报/打工等事件 随机或半随机 制造机会、不确定性、临时解法
检定结果 概率随机 制造风险与收益波动

例如浴场既是情报来源，也是随机仪式来源；BWiki 写到，浴场检定成功后可以获得不同等级情报，并开启一个随机仪式。:contentReference[oaicite:9]{index=9}

这个结构可以概括为： 

固定压力让玩家有方向；
随机机会让玩家不能只背答案；
资源成长让玩家逐渐把随机性压低。

十、路线系统：长期目标不是任务链，而是“材料清单 + 事件链”

你提到“改朝换代”“偷魔法戒指”“汇聚很多人”“合适思潮”，这个理解也对。

长线结局通常不是一个线性任务： 

A → B → C → 结局

而更像： 

路线目标：
  需要 X 人物
  需要 Y 道具
  需要 Z 声望/思潮
  需要完成若干前置事件
  需要在某个事件窗口中投入正确材料
  最后触发终局仪式

游民星空的结局指南把大路线概括为逃亡线、屠龙线、谋反线和神线，神线又可细分为正教线和密教线；并且不同结局存在结算优先级。:contentReference[oaicite:10]{index=10}

这说明它的长期结构不是“主线唯一”，而是多条终局路线并行存在。玩家在生存压力下，逐渐把手里的资源、人脉、声望、物品堆向某条路线。

十一、势力与声望：它们让选择留下长期痕迹

游戏里有多类声望，例如善名、恶名、权势、侠名、灵视。BWiki 说这些声望会触发仪式，也会影响事件能否正常进行，完成事件或随机事件会增加或减少声望。:contentReference[oaicite:11]{index=11}

声望系统的功能不是普通“阵营好感”，而是：

  • 记录你用什么方式生存；
  • 决定哪些人愿意帮你；
  • 限制或打开某些事件；
  • 把玩家从“什么都想要”推向某种路线身份。

例如玩家如果不断用暴力解决问题，可能积累恶名；如果不断帮助弱者，可能积累善名或侠名；如果经营朝堂，可能积累权势。这样一来，玩家不是在抽象地选结局，而是在长期行动中逐渐变成某种人。

十二、它的实际运转模型

如果把《苏丹的游戏》抽象成程序结构，大概是这样： 

GameState {
  day: number,
  hand: Card[],
  activeRituals: Ritual[],
  sultanCards: SultanCard[],
  characters: CharacterCard[],
  resources: ResourceCards[],
  reputations: ReputationState,
  flags: StoryFlags,
  counters: CounterState,
  fateUnlocks: MetaProgression
}

每一天的流程大概是： 

start_day:
  1. 倒计时减少
  2. 检查苏丹卡是否过期
  3. 结算到期事件 / vanish 效果
  4. 结算正在进行的仪式
  5. 根据 flags / counters / random tables 生成新仪式
  6. 玩家分配角色、物品、金币、情报
  7. 玩家决定是否结束当天

BWiki 的“下一天结算顺序”也能看出类似底层逻辑：先倒计时减少，再处理卡牌 vanish，然后仪式结算，再处理卡牌 post_rite。:contentReference[oaicite:12]{index=12}

所以它的实现并不神秘。核心其实是： 

卡牌对象系统
+ 事件/仪式系统
+ 条件触发器
+ 回合倒计时
+ 检定系统
+ 声望/flag/counter
+ 多路线结局检查

十三、为什么它会显得“混乱但有吸引力”

它给人的感觉混乱，是因为它同时运行了三种游戏：

1. 生存游戏

你必须处理苏丹卡，否则死。

2. 资源经营游戏

你要赚钱、买书、买装备、提升属性、投资家产。

3. 叙事路线游戏

你要推进人物线、势力线、逃亡线、屠龙线、谋反线、神线。

这三层不是分开的，而是互相咬合： 

苏丹卡逼你行动；
行动给你资源和后果；
资源让你能处理更难的苏丹卡；
后果把你推向某条路线；
路线又提供新的销卡方式或终局目标。

所以这个游戏的本质不是“复杂任务很多”，而是：

用一个极简单的死亡倒计时，把大量资源、人物和叙事事件压在一起。

十四、可以迁移到你自己游戏里的结构

如果你想学习它，不需要照抄苏丹卡。你要学的是它的结构。 

周期性压力目标
→ 多种解法
→ 解法需要资源
→ 资源来自事件
→ 事件消耗时间和人手
→ 时间压力迫使取舍
→ 取舍留下声望/路线后果
→ 长期路线改变最终目标

放到你的 noir 侦探游戏里，可以改成：

《苏丹的游戏》 你的 noir 游戏可替代物
苏丹卡 每隔数日出现的压力事件：罚单、威胁、委托、黑帮勒索、警局传唤
七天倒计时 几天内必须处理，否则失去钱、声誉、线索、同伴或安全
纵欲/奢靡/征服/杀戮 钱债、暴力、法律压力、人情债、情报交换、保护某人
仪式 城市地点事件：酒吧谈判、警局查档、旅馆跟踪、码头交易
随从卡 线人、记者、前同事、打手、律师、酒保
情报/思潮 证词、档案、把柄、流言、政治倾向、公众舆论
声望 警局关系、黑帮忌惮、街头名声、媒体关注、富人圈信任

你的版本不一定要“抽卡”。更适合 noir 的形式可能是： 

每隔几天，城市向科尔施加一个压力：
  - 黑帮要求他停止调查
  - 警察开罚单/传唤
  - 房东催租
  - 线人失踪
  - 委托人要求交付结果
  - 某个证人即将离城
  - 某人被威胁，需要保护

玩家必须在有限天数内处理。
处理方式可以是：
  - 付钱
  - 威胁
  - 说服
  - 找证据
  - 求人情
  - 牺牲某个关系
  - 暂时妥协
  - 转嫁给别人

最核心的理解

《苏丹的游戏》的结构可以压缩成一句话：

它用“每七天必须解决一个残酷目标”的生存压力，把资源经营、人物调度、随机事件、检定成长和多路线叙事全部绑在一起。

玩家真正玩的不是某一个任务，而是： 

在不断逼近的死亡期限前，
把世界里零碎出现的机会，
加工成足以解决当前危机和未来路线的资源。

这也是为什么它看起来混乱，但实际有一个很硬的骨架：抽卡销卡是骨架，仪式事件是肌肉，资源和人物是血液，路线结局是方向。
